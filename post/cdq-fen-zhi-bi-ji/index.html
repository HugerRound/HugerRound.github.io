cdq分治笔记

2022-02-25 5 min read # notes

cdq分治

cdq分治，就是把整个序列分成两半，计算左边的修改操作对右边查询的影响，然后对左右两个子序列重复这个过程

由于这个序列是按照时间的顺序来排列的，所以cdq分治也叫做基于时间的分治算法

正因为他是基于时间的，cdq分治可以花费log的复杂度将一个动态问题转化为一个相对简单的静态问题

所以到底是咋弄的呢

先看一个经典题目

lgP3372

线段树模板，普通区间加，区间求和

总所周知线段树是一个在线求解动态问题的算法

那么怎么离线变成静态的呢

按照上面刚才的思路

我们先把所有的操作打上一个时间标记，记为 id

同时，对于进来的每一个操作，如果是赋值，opt 记为1，反之为0

一开始整个序列的初始值可以看作在 [i,i][i,i][i,i] 这个区间内加上了 aia_iai​ ，所以也可以把他看作一个赋值操作

这 n+mn+mn+m 个操作按时间排好序（已经有序了？）后就开始重头戏——分治了

首先看到基本思想就是

void slv(int l,int r){
    if (l>=r) return ;
    int mid=(l+r)>>1;
    slv(l,mid);
    slv(mid+1,r);
}

考虑怎么在低复杂度内维护信息

看到这里面所有的修改查询都与区间有关，考虑前缀和与差分

对于 l~mid 这一段区间，我们只处理区间加操作，在差分数组上进行两端的加减

for (int i=l;i<=mid;i++){
    if (e[i].op==1){
        s[e[i].l]+=e[i].val;
        s[e[i].r+1]-=e[i].val;
    }
}

后面查询区间肯定是要利用前缀和的

所以将差分数组做两次前缀和即可

接下来当遇到查询，我们就得到查询区间的和，然后加到答案里面

for (int i=1;i<=n;i++){
    s[i]+=s[i-1];
}
for (int i=1;i<=n;i++){
    s[i]+=s[i-1];
}
for (int i=mid+1;i<=r;i++){
    if (e[i].op==2){
        ans[e[i].id]+=s[e[i].r]-s[e[i].l-1];
    }
}

问题来了：怎么证明算法的正确性呢？

其实在分治的过程中，我们已经按照二进制把某个询问前面所有修改操作分组了，有点类似线段树，故能不重不漏处理维护信息

lgP3810

三维偏序

（假装我们都会逆序对二维偏序好咯？

看上去有点难

于是一波操作直接套一个cdq分治把他转化成静态问题，多一个log

和上面那题一样，把每一个三元组变成询问和插入两个操作，方便时间上处理

这题要 ≤\leq≤ ，所以询问的优先级要低于插入

把第一维看成时间维度，首先按第一维排序，排好序以后按顺序重新将第一维打上时间戳，把第一维的范围从值域转化为时间

第二维则是在分治过程中排序，注意要开一个辅助数组记录原始按第一维排好序的数组，因为每次排序会导致原数组的变化，对其他的求解可能有影响

不难发现，在自底向上求解的过程中，两段中的数组都是按第二维排好序的，是不是归并会快一些（？

归并确实快，而且我们也可以看到，最终整个数组应该是按照第二维排好序了的

等等，这不就是归并排序？

所以从某种程度上来说，归并排序也是一种cdq分治，这也不难解释为什么归并排序可以用来解决二维偏序问题

但是下一步限制了时间复杂度只能2个log

第三维就用树状数组处理

所有时间戳小于等于mid的只处理修改，其余的只处理查询

不过注意每次清空树状数组时不能用memset，只能一个一个的减回去

这样在一次solve中，第一维所有修改的一定比查询小（mid分界线），第二维已经排好序，第三维是标准的树状数组

于是三维偏序问题就解决了

void slv(int l,int r){
	if (l>=r) return ;
	int mid=(l+r)>>1;
	slv(l,mid);
	slv(mid+1,r);
	sort(a+l,a+1+r,cmp2);
	for (int i=l;i<=r;i++){
		if (a[i].opt==1&&a[i].a<=mid){
			add(a[i].c,1);
		}
		else if (a[i].opt==2&&a[i].a>=mid+1){
			ans[a[i].id]+=query(a[i].c);
		}
	}
	for (int i=l;i<=r;i++){
		if (a[i].opt==1&&a[i].a<=mid){
			add(a[i].c,-1);
		}
	}
	for (int i=l;i<=r;i++){
		a[i]=a1[i];
	}
}
